
Assignment Overview
In this assignment, students will design and implement a TCP-based client-server application using Python Socket API. The goal is to gain practical experience with inter-process communication, socket programming, and the TCP/IP protocol stack.
Students will build both a client and a server application capable of exchanging messages and transferring files using a custom-designed application-layer protocol.
The assignment focuses on:
	•	TCP socket communication
	•	Client-server architecture
	•	Protocol design
	•	Reliable data transmission
	•	File transfer mechanisms
	•	Basic networking software development


System Requirements
Students must implement two separate programs:
	•	A TCP server
	•	A TCP client
The communication must use the Python socket library.
The system only needs to support one connected client at a time.

Functional Requirements
1. Server Requirements
The server application shall:
	•	Accept one client connection at a time
	•	Remain active until manually terminated
	•	Continue waiting for another client after a disconnect
The server must display useful status messages such as:
	•	Client connected
	•	Message received
	•	File received
	•	Client disconnected
	•	Error notifications

2. Client Requirements
The client application shall:
	•	Connect to the server using IP address and port number
	•	Allow users to enter commands through the terminal
	•	Send requests to the server
	•	Display responses from the server
	•	Disconnect gracefully from the server

Protocol Design Requirements
Students must design a simple text-based application protocol for communication between the client and server.
The protocol must support:
Operation
Description
LOGIN
Identify client/user (using an external file that lists valid users)
MSG
Send text messages
FILE
Transfer files
QUIT
Terminate connection
Students may extend the protocol with additional commands to meet the functionality requirement.

Protocol Documentation
Students must clearly document:
	•	Command syntax
	•	Message format
	•	File transfer procedure
	•	Error handling rules
	•	Connection termination procedure
Example protocol messages:
LOGIN: Alice
MSG: Hello Server
FILE: report.pdf
QUIT

Message Transfer Requirements
The application shall support text-based communication between the client and server.
Requirements:
	•	Client sends text messages
	•	Server receives and processes messages
	•	Server acknowledges received messages
	•	Messages must display correctly on both sides

File Transfer Requirements
The application must support file transfer from the client to the server.
Requirements:
	•	Transfer text or binary files
	•	Preserve original file contents
	•	Store received files in a server directory
	•	Support files of varying sizes
The file transfer process may include:
	•	Filename transmission
	•	File size transmission
	•	File content transmission

Connection Requirements
The application only needs to support a single active client connection.
Requirements:
	•	One client communicates with the server at a time
	•	Multithreading is NOT required
	•	Concurrent client handling is NOT required
The focus of this assignment is correctness and protocol implementation rather than scalability.

Error Handling Requirements
The system should handle common networking errors gracefully.
Examples include:
	•	Invalid commands
	•	Missing files
	•	Server unavailable
	•	Unexpected disconnects
	•	Empty messages
	•	Incorrect protocol format
Applications should avoid crashing whenever possible.

Technical Constraints
Required
	•	Python 3.x
	•	socket library
Allowed Libraries
	•	os
	•	time
	•	hashlib
	•	json
Not Allowed
	•	Flask
	•	Django
	•	FastAPI
	•	External networking frameworks
	•	Existing chat/file-transfer libraries

User Interface Requirements
The application may use a simple command-line interface.
Example:
> LOGIN Alice
> MSG Hello
> FILE test.txt
> QUIT
A graphical interface is optional and not required.
